# Local Todo 产品优化需求文档

> 目标：打造一款简约实用的本地化 Todo 产品，让用户在任务管理场景下可替代微软 Todo，在轻量知识管理场景下可替代语雀。

---

## 一、产品定位

```
Local Todo = 微软Todo（任务管理）+ 语雀轻量版（Markdown笔记）+ 本地优先（隐私安全）
```

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| 本地化 | 无后端、无云端依赖，数据完全掌握在用户手中 |
| 简约实用 | 功能克制，不做重型知识库，聚焦高频刚需场景 |
| 数据可控 | 完整的导入导出能力，用户随时可迁移、可备份 |

### 1.2 目标用户

- 个人知识工作者（开发者、产品经理、设计师）
- 需要同时管理任务和轻量笔记的用户
- 对数据隐私敏感、不愿依赖云服务的用户
- 微软 Todo 的重度用户，但需要更强的笔记能力
- 语雀的轻量用户（仅用于会议记录、学习笔记、技术备忘）

## 二、现状分析

### 2.1 当前能力

| 模块 | 已有能力 |
|------|----------|
| 列表管理 | 多列表创建、切换、删除、颜色标记 |
| 任务管理 | 创建、完成、删除、编辑标题/备注/日期/标签 |
| 视图 | 我的一天、重要待办、所有待办、按列表查看 |
| 排序 | 拖拽排序（组内 + 跨列表） |
| 搜索 | 标题 + 备注 + 标签全文匹配 |
| 数据持久化 | localStorage，每日自动重置"我的一天" |
| 导入导出 | JSON / XLS 格式 |

### 2.2 核心短板

| 短板 | 影响 |
|------|------|
| 无子任务 | 无法拆解复杂任务，复杂场景被迫拆成多个独立任务 |
| 无重复任务 | 周期性工作（周报、每日站会）需手动重复创建 |
| 无提醒 | 截止日期仅视觉展示，错过无感知 |
| 备注为纯文本 | 无法排版、无法插入代码/表格/链接，笔记能力弱 |
| 无暗色主题 | 夜间使用体验差 |
| 无键盘快捷键 | 效率用户操作成本高 |
| 标签孤立 | 每个任务独立维护标签，无全局视图和筛选能力 |
| 仅 localStorage | 清浏览器数据即丢失，无多设备能力 |

## 三、需求详述

### 3.1 P0 — Markdown 备注（替代语雀的核心能力）

#### 3.1.1 需求描述

将任务备注从纯文本 `textarea` 升级为 Markdown 编辑器，支持编辑和预览双模式。

#### 3.1.2 功能清单

| 功能 | 说明 |
|------|------|
| Markdown 编辑区 | 支持标准 Markdown 语法输入 |
| 实时预览 | 切换预览模式后渲染 Markdown 为 HTML |
| 工具栏 | 快捷插入：标题、粗体、斜体、列表、代码块、表格、链接、图片 |
| 代码块高亮 | 支持语言选择 + 语法高亮 |
| 图片嵌入 | 支持 base64 内嵌图片（保持本地化，不依赖外部图床） |
| 表格支持 | 标准 Markdown 表格语法 |
| 链接支持 | 外部链接 + 内部任务/文档链接（见 3.6 双向链接） |

#### 3.1.3 技术方案

| 选项 | 说明 | 推荐 |
|------|------|------|
| `@uiw/react-md-editor` | 开箱即用，编辑+预览一体 | ✅ 推荐 |
| `react-markdown` + 手动编辑区 | 灵活但需自行拼装 | 备选 |
| 自研轻量编辑器 | 仅支持子集语法，减少依赖 | 不推荐（维护成本高） |

#### 3.1.4 数据模型变更

```typescript
// 无变更，notes 字段仍为 string
// 存储 Markdown 源码，渲染时解析
```

#### 3.1.5 验收标准

- [ ] 任务详情面板中，备注区域支持 Markdown 编辑
- [ ] 预览模式下正确渲染标题、列表、代码块、表格、链接
- [ ] 代码块支持语法高亮

### 3.2 P0 — 暗色主题

#### 3.2.1 需求描述

支持浅色/暗色两种主题，可手动切换，也可跟随系统偏好自动切换。

#### 3.2.2 功能清单

| 功能 | 说明 |
|------|------|
| 主题切换 | Sidebar 底部增加主题切换按钮（太阳/月亮图标） |
| 三种模式 | 浅色 / 暗色 / 跟随系统 |
| 持久化 | 用户选择保存到 localStorage |
| 自动跟随 | 选择"跟随系统"时，监听 `prefers-color-scheme` 变化 |

#### 3.2.3 技术方案

- CSS Variables 方案：在 `:root` 和 `[data-theme="dark"]` 中定义两套变量
- 切换时仅修改 `<html>` 或 `<body>` 的 `data-theme` 属性
- 所有颜色引用必须通过 CSS 变量，禁止硬编码颜色值

#### 3.2.4 暗色主题变量定义

```css
[data-theme="dark"] {
  --sidebar-bg: #1e1e1e;
  --main-bg: #252526;
  --border-color: #3c3c3c;
  --hover-bg: #2a2d2e;
  --active-bg: #37373d;
  --text-primary: #cccccc;
  --text-secondary: #999999;
  --text-muted: #666666;
  --task-hover: #2a2d2e;
  /* 其他变量... */
}
```

#### 3.2.5 验收标准

- [ ] 切换主题后所有 UI 元素正确适配暗色
- [ ] 无硬编码颜色遗漏（包括弹窗、日期选择器、标签等）
- [ ] 刷新页面后主题保持
- [ ] 跟随系统模式下，系统主题变化时自动切换

### 3.4 P1 — 键盘快捷键

#### 3.4.1 需求描述

为高频操作提供键盘快捷键，提升效率用户的操作速度。

#### 3.4.2 快捷键清单

| 快捷键 | 功能 | 作用域 |
|--------|------|--------|
| `N` | 聚焦新建任务输入框 | 全局（输入框外） |
| `/` | 聚焦搜索框 | 全局（输入框外） |
| `1` | 切换到"我的一天" | 全局 |
| `2` | 切换到"重要待办" | 全局 |
| `3` | 切换到"所有待办" | 全局 |
| `Enter` | 打开/关闭当前选中任务详情 | 任务列表聚焦时 |
| `Delete` / `Backspace` | 删除当前选中任务（需确认） | 任务详情打开时 |
| `Esc` | 关闭任务详情面板 | 详情面板打开时 |
| `Space` | 切换当前选中任务的完成状态 | 任务列表聚焦时 |
| `I` | 切换当前任务"重要"标记 | 任务列表聚焦时 |
| `S` | 切换当前任务"我的一天"标记 | 任务列表聚焦时 |

#### 3.4.4 验收标准

### 3.5 P1 — 重复任务（Recurring）

#### 3.5.1 需求描述

支持创建周期性重复的任务，完成任务后自动生成下一个周期的实例。

#### 3.5.2 功能清单

| 功能 | 说明 |
|------|------|
| 重复类型 | 每天 / 每周 / 每月 / 每年 / 自定义间隔 |
| 自定义间隔 | 如"每 2 周"、"每 3 天" |
| 重复结束条件 | 永不结束 / 指定日期结束 / 重复 N 次后结束 |
| 自动生成 | 完成重复任务时，自动生成下一个实例 |
| 截止日期联动 | 新实例的截止日期 = 原日期 + 间隔 |
| 提醒联动 | 新实例继承原任务的提醒设置 |

#### 3.5.3 数据模型变更

```typescript
export interface Recurrence {
  type: 'daily' | 'weekly' | 'monthly' | 'yearly' | 'custom';
  interval: number;        // 间隔倍数，如每2周 = 2
  endDate: string | null;  // 重复结束日期（null = 永不结束）
  maxOccurrences: number | null; // 最大重复次数（null = 无限制）
  occurrencesCount: number;      // 已生成次数
}

export interface Task {
  // ... 现有字段
  recurrence: Recurrence | null;  // 新增
  isGenerated: boolean;            // 新增：是否为自动生成的实例
  parentTaskId: string | null;     // 新增：父任务ID（用于追踪重复链）
}
```

#### 3.5.4 交互细节

- 任务详情面板中增加"重复"设置区域
- 选择重复类型后显示对应的配置选项
- 重复任务的任务行内显示重复图标（🔄）

#### 3.5.5 验收标准

- [ ] 可创建每天/每周/每月/每年重复的任务
- [ ] 完成重复任务后自动生成下一个实例
- [ ] 新实例继承标题、备注、标签、提醒等属性
- [ ] 支持设置重复结束条件
- [ ] 导入导出时 recurrence 字段正确序列化

| 自定义时间 | 日期 + 时间选择器 |

#### 3.6.5 验收标准

### 3.10 P3 — 双向链接

#### 3.10.1 需求描述

在 Markdown 内容中支持 `[[任务标题]]` 和 `#标签` 语法，自动解析为内部链接。

#### 3.10.2 功能清单

| 功能 | 说明 |
|------|------|
| 任务引用 | `[[任务标题]]` 渲染为可点击链接，跳转到对应任务 |
| 文档引用 | `[[文档标题]]` 渲染为可点击链接，跳转到对应文档 |
| 标签引用 | `#标签名` 渲染为标签链接，点击后筛选该标签 |
| 自动补全 | 输入 `[[` 时弹出任务/文档列表供选择 |
| 反向链接 | 任务/文档详情页显示"引用了此内容的条目" |

#### 3.10.3 技术方案

- Markdown 渲染时自定义插件解析 `[[...]]` 和 `#...` 语法
- 维护一个反向索引：`{ taskId: [引用它的文档ID列表] }`
- 自动补全使用 fuzzy search 匹配标题

#### 3.10.4 验收标准

- [ ] `[[标题]]` 语法正确渲染为内部链接
- [ ] 点击链接可跳转到对应任务/文档
- [ ] 输入 `[[` 时弹出自动补全列表
- [ ] 详情页显示反向链接列表
- [ ] 被引用条目删除时，链接自动失效提示

| 列表分布 | 各列表任务数量占比（饼图/柱状图） |
| 连续打卡 | "我的一天"连续使用天数 |
| 趋势图 | 近 7 天/30 天完成任务趋势 |

#### 3.11.3 技术方案

- 使用轻量图表库（如 `recharts` 或纯 CSS 图表）
- 统计数据从 localStorage 实时计算
- 不新增持久化字段

#### 3.11.4 验收标准

- [ ] 统计面板正确展示各项数据
- [ ] 图表渲染正常，无数据时显示空状态
- [ ] 统计数据与实际任务一致

### 3.12 P3 — 数据持久化升级

#### 3.12.1 需求描述

在保持本地化的前提下，提升数据安全性和多设备可用性。

#### 3.12.2 功能清单

| 功能 | 说明 |
|------|------|
| IndexedDB 存储 | 替代 localStorage，突破 5MB 限制 |
| 自动定时备份 | 定期提示用户下载备份文件 |
| WebDAV 同步（可选） | 支持配置 WebDAV 地址，实现多设备同步 |
| 冲突解决 | 多设备同步时的冲突检测和解决策略 |

#### 3.12.3 技术方案

- IndexedDB 使用 `idb` 或 `localforage` 库封装
- WebDAV 使用 `webdav` npm 包
- 冲突解决策略：最后写入优先 / 手动选择

#### 3.12.4 验收标准

- [ ] 数据从 localStorage 迁移到 IndexedDB
- [ ] 大数据量（1000+ 任务）下读写性能正常
- [ ] 备份文件可正常导入恢复
- [ ] WebDAV 同步功能（如实现）正确同步数据

## 七、风险评估

| 风险 | 影响 | 应对 |
|------|------|------|
| Markdown 编辑器依赖体积大 | 包体积超标 | 选择轻量方案，按需加载语法高亮 |
